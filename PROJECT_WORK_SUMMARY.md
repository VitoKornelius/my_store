# Що було зроблено в цьому проєкті

Цей файл підсумовує роботу, виконану під час налаштування та розширення проєкту `my-marketplace`.

## 1. Проєкт перенесено і запущено

Після перенесення було перевірено запуск основних сервісів:

| Сервіс | Адреса |
| --- | --- |
| API | http://localhost:9000 |
| Admin | http://localhost:7000/dashboard |
| Vendor | http://localhost:7001/seller |
| Storefront | http://localhost:8000 |

Також були підняті локальні контейнери:

- `mercur-postgres` на порту `5433`;
- `mercur-redis` на порту `6379`.

## 2. Виправлено запуск admin і vendor

Під час запуску були проблеми з Vite та prebundle залежності `@medusajs/dashboard`.

Щоб admin і vendor нормально відкривалися, у Vite-конфігурації цих застосунків було додано виключення `@medusajs/dashboard` з `optimizeDeps`.

Після цього:

- admin-панель відкривається на http://localhost:7000/dashboard;
- vendor-панель відкривається на http://localhost:7001/seller.

## 3. Створено UI для покупців

У проєкт додано новий застосунок:

```text
apps/storefront
```

Це окремий React/Vite storefront для покупців.

У ньому реалізовано:

- головний екран;
- каталог товарів;
- пошук;
- фільтрацію за категоріями;
- картки товарів;
- відображення продавця товару;
- модальне вікно товару;
- вибір варіанта товару;
- локальний кошик;
- зміну кількості товарів у кошику;
- адаптивний дизайн для мобільних пристроїв.

Storefront бере дані з реального Medusa Store API через publishable API key.

Адреса storefront:

```text
http://localhost:8000
```

## 7. Створено повний seed

У `packages/api/src/scripts` додано один головний seed-скрипт для повного відновлення демо-стану.

Основна команда:

```powershell
cd packages/api
bun run seed
```

Вона створює:

- базові налаштування Medusa;
- 3 admin-акаунти;
- 12 seller-акаунтів;
- 112 товарів;
- категорії, варіанти, ціни, зображення та складські залишки;
- зв'язки товарів із конкретними продавцями.

Скрипт зроблений ідемпотентним: він перевіряє наявні акаунти й товари, тому повторний запуск не має створювати дублікати.

## 9. Перевірки

Після змін були виконані перевірки:

- storefront збирається через `bun run build`;
- backend збирається через `bun run build`;
- Store API повертає товари;
- storefront відкривається зі статусом `200`;
- продавців залишилося 12 після додавання нових товарів;
- кількість товарів збільшилася до 112;
- порти `7000`, `7001`, `8000`, `9000` слухаються.

## 10. Що ще не завершено

У storefront поки демонстраційне оформлення замовлення.

Працюють:

- каталог;
- фільтри;
- картки товарів;
- варіанти;
- кошик.

Ще потрібно підключити окремо:

- повний checkout;
- payment provider;
- email provider;
- production file storage;
- production secrets;
- production CORS.

## 11. Важливі зауваження

Демо-паролі `admin123` і `seller123` не можна використовувати в production.

Файли `.env` не потрібно додавати в Git.

Перед публічним розгортанням треба замінити:

- `JWT_SECRET`;
- `COOKIE_SECRET`;
- паролі користувачів;
- CORS-адреси;
- налаштування бази даних;
- налаштування Redis.
